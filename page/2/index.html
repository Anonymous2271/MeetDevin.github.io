图像平滑

图像平滑处理可以帮助去除图像噪音。一般是逐个对图像中单个像素点进行处理，在转换为矩阵后，即是对矩阵中单个元素进行重新赋值，根据赋值算法的不同，分为均值滤波，中值滤波等。简单来说，均值滤波即是在像素点的周围领域点取平均值，再将其赋给该像素点。

在实验中，均值滤波算法无法对边界噪音，线段型噪音进行过滤，而采用中值滤波算法则可以。本文记录下了中值滤波的实现原理，作为学习笔记。

图像模糊增强

发表于 2017-12-06 | 更新于 2018-12-23 | 评论数：

模糊增强的目的是增加图像的对比度，亮的更亮，暗的更暗，常用方法有空域法（灰度变换），频域法（滤波）。

图像的模糊特征平面

设图像 X 具有 M×N 个像素，存储深度为 L。Xmn 为图像第 (m，n) 个像素点，Pmn 为该点具有最大灰度等级的程度。
定义模糊特征变换 Pmn = G(Xmn).

G(Xmn) = (1+(Xmax - Xmn)/Fe)^(-Fd)

其中 Xmn 是像素 Xmn 的最大值，Fe 是指数型模糊因子，Fd是倒指数型模糊因子。称 Pmn(m=1,2,…,m,n=1,2,…,n)组成的平面成为图像的模糊特征平面。

避免Context内存泄漏

发表于 2017-03-14 | 更新于 2018-02-08 | 评论数：

引用 Context 对象是常有的事情，可是这里有个问题，当引用 Activity 对象的时候，必须要注意到，Activity 是有生命周期的。有时候，Activity 的生命周期已经走完了，可在项目其他地方（比如数据库操作对象等等），任然保持着该对象的引用，这里便出现了 OOM.

解决办法：

  • 控制 Activity 的周期
  • 在 Activity 中创建静态内部类，在其中创建 Context 的弱引用 WeakReference
  • 使用全局 Application 类

其中第三种方法是我最常用的，它可以为你提供一个全局的 Context 对象，而不需要关心怎样把Context对象传入进来。
