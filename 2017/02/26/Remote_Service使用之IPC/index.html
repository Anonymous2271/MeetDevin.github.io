Remote Service使用之IPC

发表于 2017-02-26 | 更新于 2018-12-23 | 评论数：

最近在写一个app，有一个计步的功能，后台服务使用 G-Sensor 来计步，一直保持运行，不受 View 影响，我决定使用 Remote Service 来完成这个任务。它会单独开启一个进程，所以如果我想在 Activity 里看到实时的步数（我一边在走路的话），就需要一个方法，后台服务能够随时调用它来更新 UI。

好在 Android 为我们提供了 AIDL (Android Interface Definition Language)，即Android接口定义语言，来公开服务的接口，进行进程间的通信。

总结

看懂上面的代码，你会发现这其实就是一个回调机制，和我之前一篇文章讲的自定义Item监听是异曲同工，Java 中的接口真是一门大学问，有时间写一系列文章好好总结总结。

再回到代码，我定义了两个接口，这其中：

  • 一个在 Activity 中创建对象，它的复写方法，是来更新 UI 的
  • 另一个在 Service 中创建对象，它的复写方法，是注册/反注册第一个接口对象的
  • Service 复写了自己的 onBind 方法，返回 Stub 第二个接口对象
  • Activity 通过绑定返回的 Stub 获得了第二个接口对象，通过 RemoteCallbackList 注册第一个接口对象
  • Service 通过 RemoteCallbackList 得到被注册的第一个接口对象，调用它的方法实现了回调
